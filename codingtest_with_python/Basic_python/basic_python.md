- 튜플 자료형 (그래프 알고리즘구현할 때 주로 사용)

  - 리스트와의 차이 (튜플 자료형은 리스트자료형과 거의 비슷)
  
    - 튜플은 한 번 선언된 값을 변경할 수 없다.
    - 리스트는 대괄호[]를 이용하지만, 튜플은 소괄호()를 이용한다.
  ```python
  a = (3,2,5,1)
  a[2] = 45
  ---------------------------------------------------------------------------
  TypeError                                 Traceback (most recent call last)
  <ipython-input-7-a8f38a1c8f84> in <module>
        1 a = (3,2,5,1)
  ----> 2 a[2] = 45

  TypeError: 'tuple' object does not support item assignment
  ```
  
  - 사용되는 경우
    - 그래프 알고리즘을 구현할 때 주로 사용됨
      e.g.) 다익스트라 최단 경로 알고리즘처럼 최단 경로를 찾아주는 알고리즘의 내부에서는 우선순위 큐를 이용하는데,
            우선 순위 큐에 한 번 들어간 값을 변경되지 않기 때문에, 우선순위 큐에 들어가는 데이터를 튜플로 구성하여 소스코드를 작성함
            -> 알고리즘 구현과정에서 혹여나 자신이 알고리즘 잘못작성해서 변경하면 안되는 값이 변경되고 있지는 않은지 체크 가능
            -> 흔히 (비용, 노드 번호)의 형태로 함께 튜플로 묶어 관리하는 것이 관례임
    - 일반적으로 각 원소의 성질이 서로 다를 때 주로 사용 ( 비용과 노드 번호처럼 서로 다른 성질의 데이터)
     
- 사전 자료형 (코테에서 자주 사용될 수 있음)

  - key와 value의 쌍을 데이터로 가지는 자료형
  - 리스트나 튜플은 값을 순차적으로 저장함 ( e.g.) 리스트 값이 [1,2,3]일 때 첫 번째 원소는 a[0]으로 1이라는 값 가짐)
  - 변경불가능한 데이터를 key로 사용할 수 있음
  - 파이썬의 사전 자료형을 이용한 효과적인 사례 -> 사전
    키로 한글 단어를 넣고 값으로 영어 단어를 넣어 사과의 영단어 알고 싶다면 사과라는 키 값을 가지는 데이터에 바로 접근하면 됨.
  - 파이썬의 사전 자료형은 내부적으로 해시 테이블을 이용함 -> 기본적으로 데이터 검색 및 수정을 O(1) 시간에 처리 가능.
  - key-value쌍으로 구성된 데이터를 처리함에 있어서 리스트 보다 훨씬 빠르게 동작함.
  ```python
  data = dict()
  data['사과'] = 'Apple'
  data['포도'] = 'Grape'
  data['오렌지'] = 'Orange
  print(data) # {'사과': 'Apple', '포도': 'Grape', '오렌지': 'Orange'}
  '
  ```
  - 코테에서 사용되는 경우
  
    - 학생의 번호가 1부터 10000000까지로 구성되어 있을 때 최대 10000명의 학생을 선택했다고 가정해보면 이후에 특정한 학생 번호가 주어졌을 때
      해당 학생이 선택되었는지를 빠르게 알 수 있는 방법은 사전 자료형을 이용하는 것(10000개의 데이터만 사전 자료구조에 들어감), 리스트를 이용
      하게 되면 1부터 10000000까지의 각 번호가 선택되었는지를 저장할 수 있는 리스트 만들어야되고 이럴 경우 999만 개 가량의 데이터는 쓰이지 않으므로
      메모리 공간이 낭비됨
      ```python
      data = dict()
      data['사과'] = 'Apple'
      data['포도'] = 'Grape'
      data['오렌지'] = 'Orange'
      
      if '포도' in data:
        print("'포도'를 키로 가지는 데이터가 존재함.")
      ```
      출력 : '포도'를 키로 가지는 데이터가 존재함.

- 집합 자료형 (집합을 처리하기 위한 자료형)

  - 특징
    - 중복 허용X
    - 순서가 없음. (인덱싱으로 값 얻을 수 없음, 리스트나 튜플은 순서가 있기 때문에 인덱싱을 통한 자료형의 값을 얻을 수 있었음)
    - key 존재X, value 데이터만 담음
    - 특정 원소가 존재하는지를 검사하는 연산의 시간 복잡도는 O(1) (사전 자료형과 동일)
    
  - 사용하면 좋은 경우
    - 학생 번호가 주어졌을 때 해당 학생이 선택되었는지 여부를 출력하는 문제 (자료형에서 언급했던)
    - 특정한 데이터가 이미 등장한 적이 있는지 여부를 체크할 때 (매우 효과적)
    
  - 초기화 (set()함수 이용 or 중괄호{}안에 각 원소를 콤마를 기준으로 넣기)
    ```python
    # 집합 자료형 초기화 방법1 (set함수 이용)
    data = set([1,3,2,5,2,3])
    print(data) # {1, 2, 3, 5}

    # 집합 자료형 초기화 방법2 (중괄호)
    data = set([1,3,2,5,2,3])
    print(data) # {1, 2, 3, 5}
    
    ```
  - 집합자료형의 연산 (합집합, 교집합, 차집합)
    ```python
    
    a = set([1,2,3,4,5])
    b = set([3,4,5,6,7])
    
    print(a | b) # 합집합 
    print(a & b) # 교집합
    print(a - b) # 차집합
    ```
    출력
    {1, 2, 3, 4, 5, 6, 7}
    {3, 4, 5}
    {1, 2}
  - 집합 자료형 관련 함수(add(), update(), remove())
    ```python
    a = set([1,2,3,4,5])

    # 새로운 원소 추가 (add()함수)
    a.add(6)
    print(a) # 출력 : {1, 2, 3, 4, 5, 6}

    # 새로운 원소 여러 개 추가 (update()함수)
    a.update([7,8])
    print(a) # 출력 : {1, 2, 3, 4, 5, 6, 7, 8}

    # 특정한 값을 갖는 원소 삭제 (remove()함수)
    a.remove(4)
    print(a) # 출력 : {1, 2, 3, 5, 6, 7, 8}
    ```

### 조건문
-> 프로그램 작성할 때 프로그램의 흐름을 제어하는 문법 (조건에 따라 로직 설정할 수 있음)

- 들여쓰기
  
  4개의 공백문자 v tab
  -> 실제 코테에서 탭 이용해도 정답 판정 받는 것에 상관X
  -> 하지만 공백문자 4개쓰는게 표준임
 
 - 파이썬의 기타 연산자
  파이썬에는 in 연산자와 not in 연산자를 제공함. 여러 개의 데이터를 담는 자료형으로 리스트, 튜플,   문자열, 사전과 같은 자료형 안에 어떠한 값이 존재하는지 확인하는 연산
  
  - a in 리스트 -> 리스트 안에 a가 들어있을 때 참(True)다.
  - a not in 문자열 -> 문자열 안에 a가 들어있지 않을 때 참(True)다.
 
 - pass
  조건문의 값이 참(True)이라도 아무 것도 처리하고 싶지 않을 때 이용
  
 - 조건부 표현식(conditional expression)
  -> if ~ else문 한 줄에 작성 가능
    ```python
    score = 98
    
    result = "Success" if score >= 90 else "Fail"
    
    print(result) # 출력 : Success
    ```
  -> 리스트에 있는 원소 값을 변경해서 또 다른 리스트 만들고자 할 때 매우 간결하게 사용가능
    ```python
    a = [1,2,2,3,4,5,5]
    remove_set = {1,2}
    result_list = [i for i in a if i not in remove_set]
    print(result_list) # 출력 : [3, 4, 5, 5]
    ```
 - 파이썬 조건문 내에서의 부등식 -> 수학의 부등식을 그대로 사용 가능
    ```python
    a = 15
    if a > 0 and a < 20:
      print("a는 0초과 20미만의 수다.")
    if 0 < a < 20:
      print("a는 0초과 20미만의 수다.")
    
    """출력
    a는 0초과 20미만의 수다.
    a는 0초과 20미만의 수다.
    """
    ```

### 함수 
  
  -> 프로그래밍에 있어 굉장히 중요
  
  -> 똑같은 코드가 반복적으로 사용되어야할 때 가 많은데 함수 사용안하면 소스코드를 매번
     일일이 작성해야되므로 소스코드가 길어지면 프로그램의 크기가 비효율적으로 커짐
  
  -> 코테에서 테스트 케이스가 입력된 뒤에 테스트 케이스만큼 특정한 알고리즘을 수행한 결과를 반복      적으로 출력하도록 하는 경우가 많음. -> 문제를 푸는 코드를 함수화하면 매우 효과적으로 풀 수        있음
    
  - global 키워드 (함수 안에서 함수 밖의 변수 데이터를 변경해야 하는 경우 시용)
    ```python
    
    a = 0
    
    def function():
      global a # global 키워드로 변수를 지정 -> 지역 변수 안만들고 함수 바깥에 선언된 변수 참조함
      a += 1
      
    for i in range(500):
      function()
    
    print(a) # 출력 : 500
    ```
  
  - 람다 표현식 (lambda express)
    
    -> 함수를 매우 간단하게 작성하여 적용가능
    
    -> 파이썬의 정렬 라이브러리를 사용할 때 정렬 기준(key)을 설정할 때에도 자주 사용됨
       6장 정렬 확인할 것
       
    ```python
    def add(a,b):
      return a + b
      
    # 일반적인 add() 메서드 사용하면
    print(add(2,6)) # 출력 :  8
    
    # 람다 표현식으로 구현한 add() 메서드 사용
    print((lambda a, b : a + b)(2,6)) # 출력 :  8
    ```

### 입출력

  - 코테에서 입력을 위한 전형적인 소스코드 (많은 문제는 공백, 줄바꿈을 기준으로 데이터를 구분)
    - 줄 바꿈 -> int(input())을 여러 번 사용 (각 데이터가 한 줄마다 입력되는 경우)
    - 공백기준으로 데이터 구분
      ```python
      # 데이터 개수 입력
      n = int(input())
      # 각 데이터를 공백으로 구분하여 입력
      data = list(map(int,input().split()))

      data.sort(reverse = True)
      print(data)
      """
      6
      1 5 70 594 6003
      [6003, 594, 70, 5, 1]
      """
      ```
      - list(map(int,input().split())
        input()으로 입력받은 문자열을 split()을 이용해 공백을 기준으로 나눈 리스트로 바꾸고
        map을 이용해 그 리스트의 모든 원소에 int()함수를 적용시킨다. 그 결과를 list()를 통해
        list로 저장한다.
      
      - 공백을 기준으로 구분하여 적은 수의 데이터 입력
        ```python
        a,b,c = map(int,input().split())
        print(a,b,c)
        """입력 출력
        2 5 4
        2 5 4
        """
        ```  
  - 입력을 최대한 빠르게 받아야 하는 경우 ( 정렬, 이진 탐색, 최단 경로 문제)
    
    -> 1,000만 개가 넘는 라인이 입력되는 경우, 압력을 받는 것만으로도 시간 초과 받을 수 있음
    
    따라서 sys 라이브러리에 정의되어 있는 sys.stdin.readline() 함수를 이용해야 됨 (input() 느림)
    
    ```python
    import sys
    sys.stdin.readline().rstrip()
    ```
    * sys 라이브러리 사용시 한 줄 입력 받고 나서 rstrip() 함수를 꼭 호출해야 됨.
    readline()으로 입력하면 입력 후 줄 바꿈 기호로 입력 되는데 그 공백 문자를 제거하기 위해 사용.
    관행적으로 외워서 사용할 것!!
    
    ```python
    import sys
    
    # 문자열 입력받기
    data  = sys.stdin.readline().rstrip()
    print(data)
    ```    
    
  - 출력 시 오류가 발생하는 소스코드 예시
    - 문자열과 숫자를 함께 출력해야 되는 경우 (단순히 더하기 연산자 이용하면 오류발생)
      ```python
      answer = 9
      print("정답은" + answer +"이다.") # TypeError: can only concatenate str (not "int") to str
      ```
    - 해결 방법
      ```python
      answer = 9
      # 1. 변수데이터를 str() 통해 문자열로 바꾸어 줌
      print("정답은 " + str(answer) + "이다.") # 출력 : 정답은 9이다.

      # 2. 각 자료형을 콤마(,)를 기준으로 구분하여 출력
      print("정답은 ", answer, "이다.") # 출력 : 정답은 9 이다.

      # 각 변수를 콤마로 구분하여 출력하는 경우, 변수의 값 사이에 의도치 않은 공백이 삽입 될 수 있음

      # 3. f-string (python 3.6 이상의 버전부터 사용가능) 단순히 중괄호{}안에 변수를 넣음으로써
          # 자료형의 변환 없이도 간단히 문자열과 숫자를 함께 넣을 수 있음.
      print(f'정답은 {answer}이다.') # 출력 : 정답은 9이다.
      ```
