### 자료형(수 자료형, 리스트 자료형, 문자열 자료형, 튜플 자료형, 사전 자료형, 집합 자료형)

- 수 자료형
  
  - 정수형
    ```
    a = 1000 # 양의 정수
    print(a)

    a = -7 # 음의 정수
    print(a)

    #0
    a = 0
    print(a)
    ```
    
  - 실수형
    ```
    # 소수부가 0일 때 0 생략
    a = 5. # 5.0
    print(a)

    # 정수부가 0일 때 0 생략
    a = -.7 # -0.7
    print(a)
    ```
    
    파이썬에서 실수형 데이터 표현하는 방식으로 e나 E를 이용한 지수 표현 방식이 있음
    코딩테스트에서 많이 사용됨
    
    e.g.) 최단 경로 문제, 큰 수 표현할 때 0의 개수가 많아 자릿수가 헷갈리는 경우(실수할 확률 적어짐)
    -> 유효숫자e^지수 = 유효숫자 x 10^지수
    ```
    a = 1e9 # 1000000000.0
    ```
    round() 함수
      ```
      """
      컴퓨터는 실수를 정확히 표현하지 못함
      왜냐하면 컴퓨터가 수 데이터 처리할 때 2진수 사용, 실수를 처리할 때 부동소수점 방식 이용, 가장 널리쓰이는 IEEE754 표준에서는 실수형 저장하기 위해 4 or 8바이트 라는 고정된 크기의 메모리를 할당하는 등의 이유 때문
      """
      a = 0.3 + 0.6 # 0.899999999999...
      ```
      따라서 소수점 값을 비교하는 작업이 필요한 문제 -> round() 함수를 이용할 것

      round(실수형 데이터, 반올림하고자 하는 위치 -1) , 인자 하나만 넣을 경우 소수점 첫째 자리에서 반올림
      e.g.) 123.456dmf 소수점 셋쨰 자리에서 반올림 하면 round(123.456, 2) 라고 작성하고 결과는 123.46 이다.

  - 수 자료형의 연산
    ```
    # 나누기
    a = 10 / 3

    # 몫
    a = 10 // 3

    # 나머지 (코테에서 많이 사용, e.g.) 특정한 변수가 홀수인지 확인할 때 (if a % 2 == 1:))
    a = 10 % 3

    # 거듭제곱 연산자
    a = 10**3 # 10^3 (1000)
    ```
    
- 리스트 자료형

  - 파이썬의 리스트 : 연결 리스트 자료구조 기능 포함하고 있어서 append()나 remove등의 메서드 지원, C++의 STL vector와 유사 
  
  - 선언방법
    ```
    # 빈 리스트 선언
    a = []
    a = list()
    ```
    
  - 초기화 (코테 문제에서는 주로 크기가 N인 1차원 리스트를 초기화 해야됨)
    ```
    # 크기가 N이고 모든 값이 0인 1차원 리스트 초기화
    n = 20
    a = [0] * n
    ```
    
  - 리스트의 인덱싱과 슬라이싱
    ```
    a = [4,6,1,3]
    
    print(a[-1]) # 뒤에서 첫 번재 원소 출력
    print(a[-3]) # 뒤에서 세 번째 원소 출력
    a[0] = 13 # 첫 번째 원소 값 변경
    print(a[1:3]) # 두 번재 원소부터 세 번째 원소까지
    
    ```
    
  - 리스트 관련 기타 메서드
    ```
    a.append() #리스트에 원소 하나 삽입
    a.sort() # 오름차순 정렬
    a.sort(reverse) # 내림차순 정렬
    a.insert(삽입할 위치 인덱스, 삽입할 값)
    a.count(특정 값) # 리스트에서 특정 값을 가지는 데이터의 개수를 셈
    a.remove(특정 값) # 특정한 값을 가지는 원소 제거 (값을 가지는 원소가 여러 개면 하나만 제거한다.)
    
    # 눈 여겨 두어야 할 것 : insert(), append(), remove()
    """
    원소의 개수가 N이면 파이썬 리스트 자료형의 append함수는 시간복잡도가 O(1)이지만 insert()의 경우 O(N)임(중간에 원소 삽입한 뒤 리스트의 원소 위치를
    조정해야되기 때문, remove도 마찬가지 이유로 O(N)이다.)
    
    -> 특정한 값의 원소를 모두 제거하려면?
    제거 하려는 모든 값에 대한 set를 만든다. target 리스트에 대해 set에 포함되어있지 않은 값만 새로운 리스트에 넣는다.
    a = [1,2,3,3,3,4,5,5,5,5,5,5]
    remove_set = {3,5}
    
    # remove_set에 포함되지 않은 값만을 저장
    result = [i for i in a if i not in remove_set) # a에 포함된 원소를 하나씩 확인하며, 그 원소가 remove_set에 포함되어 있지 않았을 때만 result에 넣음   
    """
    ```

- 튜플 자료형 (그래프 알고리즘구현할 때 주로 사용)

  - 리스트와의 차이 (튜플 자료형은 리스트자료형과 거의 비슷)
  
    - 튜플은 한 번 선언된 값을 변경할 수 없다.
    - 리스트는 대괄호[]를 이용하지만, 튜플은 소괄호()를 이용한다.
  ```
  a = (3,2,5,1)
  a[2] = 45
  ---------------------------------------------------------------------------
  TypeError                                 Traceback (most recent call last)
  <ipython-input-7-a8f38a1c8f84> in <module>
        1 a = (3,2,5,1)
  ----> 2 a[2] = 45

  TypeError: 'tuple' object does not support item assignment
  ```
  
  - 사용되는 경우
    - 그래프 알고리즘을 구현할 때 주로 사용됨
      e.g.) 다익스트라 최단 경로 알고리즘처럼 최단 경로를 찾아주는 알고리즘의 내부에서는 우선순위 큐를 이용하는데,
            우선 순위 큐에 한 번 들어간 값을 변경되지 않기 때문에, 우선순위 큐에 들어가는 데이터를 튜플로 구성하여 소스코드를 작성함
            -> 알고리즘 구현과정에서 혹여나 자신이 알고리즘 잘못작성해서 변경하면 안되는 값이 변경되고 있지는 않은지 체크 가능
            -> 흔히 (비용, 노드 번호)의 형태로 함께 튜플로 묶어 관리하는 것이 관례임
    - 일반적으로 각 원소의 성질이 서로 다를 때 주로 사용 ( 비용과 노드 번호처럼 서로 다른 성질의 데이터)
     
- 사전 자료형 (코테에서 자주 사용될 수 있음)

  - key와 value의 쌍을 데이터로 가지는 자료형
  - 리스트나 튜플은 값을 순차적으로 저장함 ( e.g.) 리스트 값이 [1,2,3]일 때 첫 번째 원소는 a[0]으로 1이라는 값 가짐)
  - 변경불가능한 데이터를 key로 사용할 수 있음
  - 파이썬의 사전 자료형을 이용한 효과적인 사례 -> 사전
    키로 한글 단어를 넣고 값으로 영어 단어를 넣어 사과의 영단어 알고 싶다면 사과라는 키 값을 가지는 데이터에 바로 접근하면 됨.
  - 파이썬의 사전 자료형은 내부적으로 해시 테이블을 이용함 -> 기본적으로 데이터 검색 및 수정을 O(1) 시간에 처리 가능.
  - key-value쌍으로 구성된 데이터를 처리함에 있어서 리스트 보다 훨씬 빠르게 동작함.
  ```
  data = dict()
  data['사과'] = 'Apple'
  data['포도'] = 'Grape'
  data['오렌지'] = 'Orange
  print(data) # {'사과': 'Apple', '포도': 'Grape', '오렌지': 'Orange'}
  '
  ```
  - 코테에서 사용되는 경우
  
    - 학생의 번호가 1부터 10000000까지로 구성되어 있을 때 최대 10000명의 학생을 선택했다고 가정해보면 이후에 특정한 학생 번호가 주어졌을 때
      해당 학생이 선택되었는지를 빠르게 알 수 있는 방법은 사전 자료형을 이용하는 것(10000개의 데이터만 사전 자료구조에 들어감), 리스트를 이용
      하게 되면 1부터 10000000까지의 각 번호가 선택되었는지를 저장할 수 있는 리스트 만들어야되고 이럴 경우 999만 개 가량의 데이터는 쓰이지 않으므로
      메모리 공간이 낭비됨
      ```
      data = dict()
      data['사과'] = 'Apple'
      data['포도'] = 'Grape'
      data['오렌지'] = 'Orange'
      
      if '포도' in data:
        print("'포도'를 키로 가지는 데이터가 존재함.")
      ```
      출력 : '포도'를 키로 가지는 데이터가 존재함.
      
  - 사전 자료형 관련 함수 (keys(), values())
  
    - keys() : key 데이터만 뽑아서 리스트로 이용
      ```
      print(data.keys()) # 출력 :  dict_keys(['사과', '포도', '오렌지'])
      ```
    - values() : value 데이터만 뽑아서 리스트로 이용
      ```
      print(data.values()) # 출력 :  dict_values(['Apple', 'Grape', 'Orange'])
      ```
      ```
      # 각 키에 따른 값을 하나씩 출력
      key_list = data.keys()
      value_list = data.values()

      for key in key_list:
        print(data[key])
      
      ```
      Apple
      Grape
      Orange
      
- 집합 자료형 (집합을 처리하기 위한 자료형)

  - 특징
    - 중복 허용X
    - 순서가 없음. (인덱싱으로 값 얻을 수 없음, 리스트나 튜플은 순서가 있기 때문에 인덱싱을 통한 자료형의 값을 얻을 수 있었음)
    - key 존재X, value 데이터만 담음
    - 특정 원소가 존재하는지를 검사하는 연산의 시간 복잡도는 O(1) (사전 자료형과 동일)
    
  - 사용하면 좋은 경우
    - 학생 번호가 주어졌을 때 해당 학생이 선택되었는지 여부를 출력하는 문제 (자료형에서 언급했던)
    - 특정한 데이터가 이미 등장한 적이 있는지 여부를 체크할 때 (매우 효과적)
    
  - 초기화 (set()함수 이용 or 중괄호{}안에 각 원소를 콤마를 기준으로 넣기)
    ```
    # 집합 자료형 초기화 방법1 (set함수 이용)
    data = set([1,3,2,5,2,3])
    print(data) # {1, 2, 3, 5}

    # 집합 자료형 초기화 방법2 (중괄호)
    data = set([1,3,2,5,2,3])
    print(data) # {1, 2, 3, 5}
    
    ```
  - 집합자료형의 연산 (합집합, 교집합, 차집합)
    ```
    
    a = set([1,2,3,4,5])
    b = set([3,4,5,6,7])
    
    print(a | b) # 합집합 
    print(a & b) # 교집합
    print(a - b) # 차집합
    ```
    출력
    {1, 2, 3, 4, 5, 6, 7}
    {3, 4, 5}
    {1, 2}
  - 집합 자료형 관련 함수(add(), update(), remove())
    ```
    a = set([1,2,3,4,5])

    # 새로운 원소 추가 (add()함수)
    a.add(6)
    print(a) # 출력 : {1, 2, 3, 4, 5, 6}

    # 새로운 원소 여러 개 추가 (update()함수)
    a.update([7,8])
    print(a) # 출력 : {1, 2, 3, 4, 5, 6, 7, 8}

    # 특정한 값을 갖는 원소 삭제 (remove()함수)
    a.remove(4)
    print(a) # 출력 : {1, 2, 3, 5, 6, 7, 8}
    ```
